研究路线与实验设计 / 2026-07-20

大语言模型辅助电力系统优化的技术路线与实验设计

公开电网数据构成可复用的研究基础，但 LLM 对数值求解过程的增量价值仍需通过严格对照验证。建议将三类数据对齐至统一、可执行、可比较且可复验的实验流水线。

近期研究主线 完成 RTS-GMLC 数据接入与无 LLM 正式实验矩阵，同时复现两项公开的大语言模型电力优化基线。

正式矩阵用于评估冲突诊断任务是否存在可利用的策略空间；公开基线用于校准数据、任务定义与评价协议。两者共享统一的求解器内核与验证器。

03

候选研究方向与优先级

每项候选方向均需与静态规则、轻量学习方法及调优后的数值求解器进行系统比较。

A
优先研究方向

模型修订与隐藏验证

由 LLM 根据自然语言修订 OPF/UC 模型，并利用未向 LLM 暴露的反事实场景与物理 checker 识别“可执行但语义不一致”的假成功。

数据
ProOPF + PGLib-OPF + RTS-GMLC
核心指标
false-success rate、隐藏测试通过率
主要风险
基础假成功率过低，或隐藏测试缺乏方法区分度
B
当前验证方向

冲突诊断与证书压缩

AI 负责选择检查时间窗、约束族或提取配置；求解器负责生成并复验不可行证书，LLM 不参与安全约束松弛决策。

数据
UnitCommitment.jl + RTS-GMLC
核心指标
证书大小、求解次数、总时间、跨网架迁移
主要风险
单一静态策略已接近最优，或原生 IIS 提取开销不足
C
第二阶段

参数选择与暖启动

根据实例特征选择 MIP 参数、变量固定或上一时刻解的迁移方式；求解器始终保留最终可行性与最优性证明。

数据
大量同分布 UC/SCUC 时序实例
核心指标
端到端墙钟时间、最优间隙、失败回退率
主要风险
轻量模型已达到同等效果，LLM 推理开销无法摊销
D
非首篇优先方向

LLM 直接生成调度方案

由 LLM 直接生成机组启停、出力或潮流结果。该方向具有较强展示性，但在硬约束满足、大规模扩展及商业求解器对照方面风险较高。

数据
PowerGraph-LLM、经济调度小算例
核心指标
可行率、最优差距、推理成本
主要风险
安全性差、可扩展性弱、baseline 强
04

统一对照体系与证据标准

所有候选方向采用一致的对照层级、成本核算与统计标准，以识别 LLM 相对于传统方法的独立增量。

0求解器基线默认配置 + 调优配置
1固定规则人工启发式、随机策略
2轻量学习器LightGBM / bandit / 小策略网络
3通用 LLM计入 API 延迟、token 与失败
4Skill learning仅在 LLM 确有增量后加入
SkillOpt 仅在基础增量证据成立后纳入。

需依次证明任务具有非平凡难度、最优策略具有实例条件性、中间反馈具有决策价值，且 LLM 优于轻量学习方法；在此基础上，validation-gated skill learning 才构成可辨识的方法贡献。

05

近期实施计划与阶段闸门

将数据建设、基线复现与论文决策组织为具有明确交付物和终止条件的顺序阶段。

  1. 数据接入与治理完成 RTS-GMLC 转换、admission 检查和语义覆盖；整理三类数据的下载清单与许可证。交付：统一数据目录 + provenance 清单
  2. 预注册裁决实验完成冲突提取 10×3 正式矩阵，评估效应量、实例条件性与可摊销计算时间。交付：GO/NO-GO 报告
  3. 公开 LLM 基线复现以 LLM_PowerSystems 验证最小闭环，以 ProOPF-B 复现自然语言建模 benchmark。交付：可重复运行的 baseline harness
  4. 首篇方向决策若冲突诊断的策略改进空间得到复现，则继续 GridConflictBench；否则启动“模型修订与隐藏验证”先导实验。交付：冻结后的论文问题、指标与实验矩阵
  5. 策略学习（条件阶段）先评估 LightGBM，再引入通用 LLM；仅当 LLM 增量成立时加入 SkillOpt 式 validation gate。交付：分层、可解释的增量证据
研究结论

数据底座是确定性资产，LLM 切入点是待验证假设。

当前应将公开论文数据与传统求解器基准接入统一 verifier；首篇论文聚焦冲突诊断或模型修订，应依据正式矩阵与预注册 pilot 的结果确定。
