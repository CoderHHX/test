04 / 适用场景与求解器边界

虚拟电厂、负荷转供与源荷预测如何使用求解器

三个场景都涉及计算，但求解器的位置不同：虚拟电厂和负荷转供需要从大量方案中做受约束决策；源荷预测主要负责产生未来输入，只有在一致性修正或连接下游调度时才进入优化闭环。

判断原则 预测一个数，不一定需要优化求解器；从多种方案中选出满足安全与成本约束的方案，通常需要。
模型训练机器学习优化器

Adam、SGD 等，用来更新预测模型参数，不负责生成电网运行方案。

物理校验潮流计算工具

OpenDSS、pandapower 等，用来计算电压和线路潮流，判断方案能否物理运行。

方案决策数学优化求解器

HiGHS、Gurobi、COPT、CPLEX、IPOPT 等，从可行方案中选择成本或安全性更优的方案。

01
Virtual Power Plant

虚拟电厂

求解器是核心
预测、电价、设备状态→优化求解器→报价与设备计划→实时滚动修正
通常不用求解器

设备监控、数据汇总、单纯预测风光或负荷。

需要求解器

日前报价、储能充放电、EV 调度、需求响应、备用与实时控制。

求解器决定
每台设备何时发电、充电、放电、移峰，以及市场申报多少电量。
关键约束
功率平衡、储能 SOC、设备上下限、用户舒适度、合同与备用要求。
常见模型
LP / MILP、随机优化、鲁棒优化、滚动时域 MPC。
与当前能力
可复用 RTS-GMLC 时序、UC 约束和 HiGHS；还需补储能、EV 与市场规则。

02
Load Transfer

负荷转供

优化 + 物理复验
故障、拓扑、负荷→开关候选动作→优化求解器→潮流安全复验
通常不用求解器

查询固定转供预案、人工按规则操作，或只对既定方案做潮流计算。

需要求解器

自动选择开关组合、最小停电范围、最少操作次数和重要负荷优先恢复。

求解器决定
哪些开关断开或闭合、多少负荷转移、是否切负荷及分布式电源如何支撑。
关键约束
辐射状拓扑、线路容量、电压范围、功率平衡、N-1 与负荷优先级。
常见模型
MILP、MISOCP、MINLP，并由三相潮流工具复验。
与当前能力
需新增配电网、联络开关和三相潮流 verifier；现有输电 PTDF 不能直接替代。
代表性研究 Toward a MILP Modeling Framework for Distribution System Restoration IEEE Transactions on Power Systems · 2019 · OpenAlex 引用 140
03
Source–Load Forecasting

源荷预测

求解器按需使用
历史数据与天气→预测模型→预测结果→可选：优化求解器一致性修正 / 下游调度
纯预测：不用求解器

流程到“预测结果”即结束，直接输出未来负荷、风电或光伏的点预测、区间预测和概率分布。

决策衔接：按需使用求解器

只有做区域/全网预测一致性修正（QP），或将预测接入 UC、VPP、转供等调度任务时才调用。

求解器决定
修正不一致预测，或把预测转成机组启停、经济调度与储能计划。
关键约束
分区加总一致、合理上下界，以及下游调度中的设备与网络约束。
常见模型
预测修正常用 QP；下游决策使用 LP / MILP；可扩展到 Decision-Focused Learning。
与当前能力
RTS-GMLC 已含负荷、风电、光伏时序，是三者中接入现有数据基座最快的场景。
代表性研究 Using Quadratic Programming to Optimally Adjust Hierarchical Load Forecasting IEEE Transactions on Power Systems · 2018 · OpenAlex 引用 6
与当前项目的衔接顺序数据复用难度与业务闭环强度并不相同
  1. 1最快接入：源荷预测 → UC/SCUC直接复用 RTS-GMLC，以最终调度成本评价预测，而不只看平均绝对误差（MAE）。
  2. 2完整闭环：虚拟电厂调度在现有调度内核上增加储能、柔性负荷和市场报价。
  3. 3物理安全约束最强：负荷转供单独建设配电网开关模型与三相潮流 verifier，工程投入最大。
